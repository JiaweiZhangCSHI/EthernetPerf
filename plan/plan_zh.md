# 执行计划（中文）：CT 原始数据接收扩展

## 1. 目标
基于当前 Phase 1（10GbE）成果，推进后续工作到近期 100G 能力，并保留向专用采集卡与 200G 路线升级的受控决策路径。

## 2. 范围
本轮纳入：
- 10G 早期冒烟稳定与后期资格验收闭环。
- 100G 方案评估与验证攻关。
- 定量对比与 go/no-go 门槛决策。
- 供应商能力与集成风险评估。

本轮不纳入：
- 立即落地 RDMA。
- 完整自研生产级驱动开发。
- 生产发布级安全加固。

当前拓扑假设：
- 当前验证场景仅采用 FPGA 与 recon PC NIC 直连。
- 交换机拓扑不属于当前基线范围，只能作为后续扩展或对照场景处理。

## 3. 背景：关键因素与技术说明
- PPS 与小包压力：在 100G 场景下，小包业务往往先触发包率上限，再触发链路吞吐上限。
- CPU 与 NUMA 拓扑：线程绑核与网卡队列亲和性需匹配 NUMA 结构，避免跨节点访问开销。
- Windows socket 开销：WinSock UDP 在高包率下存在每包处理开销，容易压缩 CPU 可用余量。
- MTU 与 Jumbo 影响：Jumbo 帧可降低 PPS 压力与单位字节处理开销，但需端到端链路一致支持。
- 可观测性深度：队列级丢包、时间戳精度与乱序统计是定位瓶颈的基础能力。
- 集成与驱动成熟度风险：不同厂商在 Windows 驱动与 SDK 完整性上差异明显，直接影响交付风险。
- 成本与锁定风险：SmartNIC 或专用卡可提升性能上限，但也会提高长期成本与迁移难度。
- 200G 扩展路径：100G 阶段的架构决策应保留 200G 演进空间，避免后续整体重构。
- NIC DMA 能力：网卡直接内存传输能力会显著影响 CPU 占用和持续接收吞吐。
- NIC 队列与中断机制：RSS/多队列、环形缓冲深度和中断合并是首要调优抓手。
- NIC 硬件卸载能力：校验和卸载、RSC/LRO、硬件时间戳会影响效率与可观测性质量。

## 4. 路线图与阶段

### 阶段 0.5：当前 10G 早期冒烟与后期资格验收（最高优先）
1. 固化 Windows UDP 在 10GbE 下的稳定接收基线。
2. 达成双阶段验收基线：
- 早期冒烟门槛：持续有效吞吐 >= 8 Gbps，运行时长 >= 60 秒，且连续 3 次通过；
- 后期资格门槛：按 CT 业务模型假设，以单次扫描成功率置信方法验证（10 年使用期、约 300 人次/天、单次扫描约 10 秒原始数据突发；折算日均有效采集时长约 3000 秒）。
3. 固化基线交付：
- 最终 KPI 输出模板；
- 验收负载下 CPU 基线；
- 包长分布与 jumbo 观测标志；
- 调优记录（网卡参数、socket 缓冲、绑核策略）。
4. 增加分阶段丢包定位流程与证据包：
- sender/直连链路/NIC/UDP/app 五层 1 秒级同步计数快照，
- 分级负载爬坡（2G -> 4G -> 6G -> 8G），
- 基于当前直连拓扑的 A/B 隔离验证（基线驱动 vs 调优驱动、基线 NIC 参数 vs 调优 NIC 参数），
- 输出首次计数分歧点与归属结论。
5. 每次验收运行补充 Windows 命令采集记录：
- `Get-NetAdapterStatistics -Name "<NIC_NAME>"`，
- `Get-NetAdapterAdvancedProperty -Name "<NIC_NAME>"`，
- `netstat -s -p udp`，
- `netstat -s -p ip`，
- `Get-Counter '\Network Interface(*)\Packets Received Discarded','\Network Interface(*)\Packets Received Errors','\Processor(_Total)\% Processor Time' -SampleInterval 1 -MaxSamples 60`。

进入阶段 1 条件：
- 10G 冒烟门槛结果可复现；
- 后期资格验收的单次扫描置信证据完整（p0 = 99.50%，95% 置信度）；
- 测量链路可信；
- 基线调优参数已文档化；
- 若出现计数不一致，已完成首次分歧阶段定位并形成证据；
- 在 8 Gbps 目标运行下不存在未归因的计数差异；
- 后期资格验收中已证明 CPU <= 30%。

### 阶段 1：100G 规划与采购准备
1. 建立三条候选路线清单：
- 标准高速以太网网卡；
- SmartNIC/采集加速卡；
- FPGA/自定义专用采集卡。
2. 形成厂商能力矩阵：
- Windows 驱动成熟度；
- SDK/API 稳定性；
- 时间戳与可观测性能力；
- 交付周期与支持模式；
- DMA 与 RSS/多队列能力；
- RX/TX 环形缓冲可调能力与中断合并控制项；
- 校验和卸载、RSC/LRO、Jumbo 支持与硬件时间戳支持情况。

### 阶段 2：100G 验证与 A/B 基准测试
1. 按优先级执行：
- A 路：先以太网；
- B 路：门槛失败后升级 SmartNIC/采集卡；
- C 路：仅在路线图级确定性要求下评估专用卡。
2. 负载覆盖：
- 大包负载；
- 混合负载；
- 小包高包率压力负载。
3. 指标采集：
- 持续有效吞吐；
- 丢包估计与乱序计数；
- CPU 利用率与余量；
- pps 与队列级可观测性；
- 与后期 Phase 0.5 一致的单次扫描置信证据。

### 阶段 3：选型与升级决策
1. 若门槛稳定通过，保持以太网主路线。
2. 若调优后仍连续失败，升级 SmartNIC/采集卡评估。
3. 仅当必须满足更强确定性高吞吐时，评估 FPGA/自定义专用卡。
4. 当 Windows 优化余量不足时，Linux 作为可接受升级平台。

## 5. 100G 对比表

| 方案 | 预期吞吐（100G目标） | 抗包率压力能力 | CPU 开销风险 | Windows 集成风险 | 实施工作量 | 成本等级 | 建议 |
|---|---|---|---|---|---|---|---|
| 标准高速以太网网卡 + UDP 调优 | 中高到高 | 中等（小包 PPS 敏感） | 中高 | 低到中 | 低到中 | 低到中 | 近期 100G 主路径 |
| SmartNIC / 采集加速卡 | 高 | 高 | 中等 | 中高（依赖 SDK） | 中高 | 高 | 以太网路线连续失败后的第一升级路径 |
| FPGA/自定义专用采集卡 | 很高 | 很高 | 低到中（协同设计质量高时） | 高 | 很高 | 很高 | 必须追求确定性高吞吐时采用 |

## 6. 100G 加权评分（决策支持）

| 维度 | 权重 | 以太网 | SmartNIC/采集卡 | FPGA/专用卡 |
|---|---:|---:|---:|---:|
| 集成容易度 | 15 | 5 | 3 | 2 |
| Windows 成熟度与支持 | 20 | 5 | 3 | 2 |
| 交付速度 | 15 | 5 | 3 | 1 |
| 成本与锁定风险 | 15 | 5 | 3 | 2 |
| 100G 吞吐潜力 | 20 | 4 | 5 | 5 |
| CPU 效率 | 10 | 3 | 5 | 5 |
| 可观测性深度 | 5 | 4 | 4 | 2 |
| 总分 | 100 | 92.00 / 4.60 | 73.00 / 3.65 | 55.00 / 2.75 |

解读：
- 近期 100G 目标优先建议以太网路线，因其集成速度和 Windows 成熟度显著降低交付风险。
- 仅在调优后仍连续门槛失败时，才升级到 SmartNIC/采集卡路线。

## 7. 验收门槛模板
1. 吞吐门槛：整个验收窗口内持续有效吞吐达标。
2. 质量门槛：丢包/乱序低于定义阈值。
3. 生命周期统计门槛：以后期单次扫描成功率置信方法作为主门槛。
- Phase 0.5 后期默认目标：单次扫描成功率 p0 = 99.50%，
- Phase 0.5 后期默认置信度：95%，对应零失败证据至少 598 个窗口，
- 100G 资格默认置信度：95%，对应零失败证据至少 29956 个窗口，
- 可选更强发布证据：99% 置信度且至少 46050 个零失败窗口。
单日/单周/10000 窗口测试可继续作为工程趋势监控，但不能替代主置信门槛。
4. 效率门槛：CPU 平均使用率 <= 30%。
5. 稳定性门槛：连续多轮可复现通过。

## 8. 治理与交付物
1. 每周检查包：
- 测试进展；
- 阻塞与缓解；
- 供应商/采购状态；
- 下周动作。
2. 里程碑评审包：
- 三路线对比摘要；
- 加权评分更新；
- go/no-go 建议。
3. 最终决策包：
- 选型结论；
- 风险台账；
- 后续实现阶段的排期与资源估算。
